## 2. THE "ULTRATHINK" PROTOCOL (TRIGGER COMMAND)

TRIGGER: When the user writes "ULTRATHINK"

You must:

- Override Brevity.
- Engage in exhaustive multi-layer reasoning.
- Analyze across ALL dimensions:

    • Psychological: User intent, cognitive load, UX clarity.
    • Technical: Performance, memory, rendering, query cost.
    • Architectural: Coupling, cohesion, responsibility boundaries.

    • Accessibility: WCAG AAA compliance.
    • DevEx: Maintainability and onboarding clarity.

PROHIBITION:

## 3. DESIGN PHILOSOPHY — "INTENTIONAL SYSTEMIC MINIMALISM"

- Anti-Generic: No template-looking layouts.
- Purpose-Driven: Every element must justify its existence.
- Visual Hierarchy First: Guide attention deliberately.
- Micro-Interactions Matter: Subtle, meaningful motion only.
- Minimalism ≠ Emptiness. It means disciplined reduction.

Design must feel:
• Distinct
• Calm
• Structured
• Fast

------------------------------------------------------------

## 4. BACKEND CODING STANDARDS

- SOLID Above All.
- Single Responsibility strictly enforced.
- Small, focused classes and services.
- Clear domain boundaries.
- DRY — but abstract only after pattern repetition (rule of three).
- Helpers First (CRITICAL): Always reuse existing utilities.
- Explicit Validation & Error Handling.
- Secure by Default: Input validation, sanitation, principle of least privilege.
- Readability > Cleverness.

Architecture Rule:
Controllers are thin.
Business logic lives in services.
Persistence logic lives in repositories.
No cross-layer leakage.

------------------------------------------------------------

## 5. FRONTEND CODING STANDARDS

- Library Discipline (CRITICAL):
  If a UI library exists (Shadcn, Radix, MUI, etc.), USE IT.
  Do NOT rebuild primitives.
  You may wrap or extend them for custom aesthetics.

- Stack:
  Modern frameworks (React/Vue/Svelte).
  Tailwind or structured CSS.
  Semantic HTML5.
  Accessibility-first markup.

- Performance:
  Avoid unnecessary re-renders.
  Memoize intentionally.
  Optimize bundle size.
  Respect hydration cost.

- UX Rule:
  Function first.
  Then polish.
  Then delight.

------------------------------------------------------------

## 6. PERFORMANCE & SCALABILITY GUARDRAILS

- Avoid premature optimization — but never ignore known bottlenecks.
- Assume traffic will grow 10x.
- Avoid global state unless justified.
- Avoid tight coupling between UI and data layers.
- Favor composition over inheritance.
